# Help Desk Activities

## HD-0003 – No Internet Access

**Priority:** High

**Department:** Finance

A user reported that their workstation displayed **"No Internet Access"** and was unable to access websites or cloud services. A structured troubleshooting process was followed to determine the cause of the connectivity issue.

### Investigation Performed

- Verified Wi-Fi connection.
- Reviewed IP configuration using `ipconfig`.
- Verified DHCP and DNS settings using `ipconfig /all`.
- Tested internet connectivity with `ping 8.8.8.8`.
- Tested DNS resolution with `ping google.com`.
- Verified DNS functionality using `nslookup`.
- Reviewed network adapter configuration.

### Root Cause

The workstation received an **Automatic Private IP Address (APIPA)** after failing to obtain a valid IP address from the DHCP server.

### Resolution

- Released the current DHCP lease.
- Requested a new DHCP lease.
- Verified a valid IP address was assigned.
- Confirmed DNS and gateway configuration.
- Verified internet connectivity was restored.
- Confirmed successful access to company resources and websites.

**Ticket Status:** Resolved

# Challenges Encountered

Understanding the difference between internet connectivity and DNS resolution required careful analysis of command output. Comparing the results of `ping 8.8.8.8` and `ping google.com` demonstrated how networking tools can isolate DNS issues from general connectivity problems.

---

# Lessons Learned

- A valid IP address is essential for network communication.
- DHCP automatically provides network configuration information.
- DNS translates domain names into IP addresses.
- `ping` and `nslookup` answer different troubleshooting questions and should be used together when diagnosing connectivity issues.
- Following a structured troubleshooting process improves accuracy and reduces resolution time.
